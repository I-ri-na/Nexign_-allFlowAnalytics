# CallFlowAnalytics 📞

## 📌 О проекте
**CallFlowAnalytics** — это RESTful микросервис, разработанный для эмуляции работы телефонного коммутатора и аналитики биллинговых данных. Приложение генерирует детализированные записи о вызовах (CDR) и предоставляет API для получения агрегированной информации (UDR) и выгрузки файлов отчетов.

## 🛠 Стек технологий
* **Язык:** Java (OpenJDK 17)
* **Фреймворк:** Spring Boot
* **База данных:** H2 Database (in-memory)
* **Сборка:** Maven / Gradle

## 🚀 Основной функционал

### 1. Эмулятор коммутатора (Генерация CDR)
* Автоматическая генерация CDR-записей для пула абонентов (от 10 пользователей).
* Имитация реального потока звонков: данные генерируются в строгом хронологическом порядке, звонки разных абонентов пересекаются.
* Длительность и количество звонков определяются случайным образом.
* За один прогон система генерирует объем данных, эквивалентный одному году работы коммутатора.

### 2. REST API для биллинга (UDR)
* Выгрузка UDR-записи по конкретному номеру абонента (за календарный месяц или за весь тарифицируемый период).
* Выгрузка массива UDR-записей по всем абонентам базы за запрошенный месяц.
* Вся аналитика строится строго на основе данных, сохраненных в БД H2.

### 3. Асинхронная генерация отчетов
* REST-метод для инициации формирования выписки по звонкам за любой кастомный период (например, 2 недели или 6 месяцев).
* Метод работает асинхронно: возвращает статус (успех/ошибка) и уникальный `UUID` запроса.
* Отчеты физически сохраняются в директории `/reports` в форматах `.csv` или `.txt`.
* Формат нейминга файлов: `{номер_телефона}_{UUID}.csv`.
